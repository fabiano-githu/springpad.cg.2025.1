# Projeto ONCredit - Descrição do Projeto

_`Por Luferat em 30/01/2026`_

## 1. Visão Geral

O **ONCredit** é um aplicativo mobile/web desenvolvido em **Flutter** para controle simples de vendas fiado (credário informal) para pequenos comerciantes. O foco é usabilidade, simplicidade e operação por **usuário único**, sem login tradicional complexo.

O aplicativo resolve o controle manual feito em cadernos, permitindo:

* Cadastro de clientes
* Registro de compras (dívidas)
* Registro de pagamentos (abatimentos)
* Visualização de saldo devedor e histórico



## 2. Público-Alvo

* Pequenos comerciantes
* Uso individual ou por pessoas de confiança
* Usuários com pouca familiaridade com computador, mas com uso frequente de celular



## 3. Objetivos do Projeto

* Aprender Flutter de forma prática
* Criar um MVP funcional e utilizável no mundo real
* Evitar soluções complexas de autenticação
* Focar em CRUD simples e confiável



## 4. Escopo Funcional

### 4.1 Funcionalidades Principais

* Autenticação de Aplicativo, não de usuário
* Cadastro de clientes
* Registro de compras por cliente
* Registro de pagamentos por cliente
* Cálculo automático de saldo (compras − pagamentos)
* Resumo do cliente selecionado

### 4.2 Funcionalidades Futuras (opcionais)

* Desenvolver a autenticação por usuário
* Backup/restore explícito
* Exportação de dados
* Impressão de relatórios



## 5. Arquitetura Técnica

### 5.1 Frontend

* Flutter (Android e Web)
* Navegação por rotas
* Interface simples, botões grandes e poucos fluxos

### 5.2 Backend

* Firebase Realtime Database (RTDB)
* Acesso via REST (Dio)
* Estrutura de dados organizada por UID de aplicativo



## 6. Estrutura de Dados (RTDB)

```json
users/{uid}/
  ├── clients
  ├── purchases
  └── payments
```

### 6.1 Cliente

* name
* cpf
* phones[]
* createdAt

### 6.2 Compra (Dívida)

* clientId
* description
* quantity
* unitValue
* totalValue
* date

### 6.3 Pagamento

* clientId
* value
* method
* date

> O saldo **não é armazenado**, é calculado no aplicativo.



## 7. Segurança (RTDB Rules)

* Cada usuário acessa apenas seus próprios dados
* Baseado no UID do aplicativo instalado



## 8. Stack de Dependências

```yaml
flutter
dio
intl
```



## 9. Páginas do Aplicativo

### 9.1 Home

* Seleção de cliente
* Resumo financeiro
* Ações principais (nova compra, novo pagamento, extrato)

### 9.2 Cliente

* Cadastro
* Edição
* Listagem

### 9.3 Compra

* Cadastro de nova compra
* Vinculada ao cliente selecionado

### 9.4 Pagamento

* Registro de pagamento
* Abatimento da dívida

* Texto legível
* Ícones claros
* Uso consistente de cores
